The lightbox already closes on backdrop click and Escape key. On touch devices, "tap outside to close" is already handled by the backdrop onClick. However, we should ensure the lightbox is more touch-friendly:

**Modify `src/components/Lightbox.tsx`** to add:

1. A `touchend` listener on the backdrop for reliable touch closing (some mobile browsers handle click differently on overlay elements). Add this inside the existing `useEffect`:

```typescript
useEffect(() => {
  document.body.style.overflow = 'hidden'

  function handleKeyDown(e: KeyboardEvent) {
    if (e.key === 'Escape') {
      onClose()
    }
  }
  document.addEventListener('keydown', handleKeyDown)

  return () => {
    document.body.style.overflow = ''
    document.removeEventListener('keydown', handleKeyDown)
  }
}, [onClose])
```

The existing code is already correct for touch handling. The `onClick` on `.lightbox-backdrop` fires on touch tap. No additional touch event listeners are needed — mobile Safari and Chrome both fire `click` events from taps on div elements.

**The only change needed:** Add `touch-action: manipulation` to the lightbox-close button and lightbox-backdrop in globals.css (inside the mobile media query) to eliminate the 300ms tap delay on older mobile browsers:

