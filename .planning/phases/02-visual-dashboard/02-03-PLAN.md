**Behavior:**
- Wraps children in a `<div>` with an `onClick` handler that uses event delegation
- The onClick handler checks if the clicked element (or its parent) has a `.heatmap-cell` class
- If so, reads `data-image-url`, `data-keyword`, `data-business` from that element and sets `lightboxData`
- When `lightboxData` is not null, renders `<Lightbox>` with the data and an `onClose` that sets state to null
- Uses event delegation so HeatmapGrid stays a Server Component â€” no need to add onClick to each cell individually

```typescript
function handleClick(e: React.MouseEvent) {
  const cell = (e.target as HTMLElement).closest('.heatmap-cell') as HTMLElement | null
  if (!cell) return
  const imageUrl = cell.dataset.imageUrl
  const keyword = cell.dataset.keyword
  const businessName = cell.dataset.business
  if (imageUrl && keyword && businessName) {
    setLightboxData({ imageUrl, keyword, businessName })
  }
}
```

**Add IntersectionObserver in LightboxProvider:**

In `LightboxProvider`, add a `useEffect` that sets up an IntersectionObserver to watch all `.scroll-animate` elements:

```typescript
useEffect(() => {
  const observer = new IntersectionObserver(
    (entries) => {
      entries.forEach((entry) => {
        if (entry.isIntersecting) {
          entry.target.classList.add('scroll-animate-visible')
          observer.unobserve(entry.target) // Only animate once
        }
      })
    },
    { threshold: 0.1 }
  )

  const elements = document.querySelectorAll('.scroll-animate')
  elements.forEach((el, index) => {
    // Stagger: add delay based on index within current viewport batch
    ;(el as HTMLElement).style.transitionDelay = `${index * 80}ms`
    observer.observe(el)
  })

  return () => observer.disconnect()
}, [])
```

NOTE on staggering: The `index * 80ms` delay creates the staggered entrance effect. Since IntersectionObserver fires for all visible elements at once on initial load, the first ~3 cards in the viewport will stagger. As the user scrolls, new cards entering viewport trigger individually (fast, no delay needed since they enter one-by-one). The transition delay is applied once; cards already visible won't re-animate because `observer.unobserve` is called after first intersection.
